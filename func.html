get(entity: str, query: str, fields: str, count: int, start: int, BHtoken: str)[source]#

Accessing data via the API and ensuring the BHToken is always correct

Parameters

  • entity (str) – Specifies the subject that we want to extract information about.

  • query (str) – Lucene query string. Applies to Candidate and Note entities only.

  • fields (str) – Comma-separated list of field names.

  • count (int) – Limit on the number of records to return. If the set of matched results is larger than count, cap the returned results at size count.

  • start (int) – From the set of matched results, return record numbers start through (start + count)

  • BHtoken (str) – Token that represents a session established by the login process. Must be sent with all subsequent requests to the API. The session key can be provided in the BhRestToken query string, a cookie, or an HTTP header.

Raises

Exception – returns the status code for the reason of failure(reference the Bull horn Rest API doc)

Returns

returns a JSON file

Return type

Json_file

User Guide#

Introduction#

This documentation provides a comprehensive overview of the Bullhorn API, including links to important contacts and other useful documentation. The following structure to this documentation is as follows:

  1. Authentication credentials

  2. Python scripts

    1. Authentication script

    2. API data extraction script

    3. Updater script

    4. Additional scripts

  3. Additional information

Authentication credentials#

The Authentication credentials are separate credentials to the normal Bullhorn credentials needed to log onto the GUI, and must be obtained from the Bullhorn account manager. The credentials can be requested through edgaete@harnham.com. Different Bullhorn accounts will require different Auth credentials (ie. External recruitment Bullhorn, Internal recruitment Bullhorn and Rockborne Bullhorn store data on different instances of Bullhorn and will all have different credentials). The following credentials are required:

  • Client ID

  • Client secret

  • API username

  • API password

The obtained credentials should subsequently be stores as environment variables (either via Windows Command Prompt using setx or via a Python IDE such as Pycharm). Environment variables are used to keep passwords discrete, especially if code is shared on Github.

Python scripts#

This section provides a run-through of the Python scripts that are vital to the running of the API. Authentication script (auth.py) This script is run first as it uses an automated web browser to extract a Rest token (BHtoken) - which provides and maintains access to the API (think of a key which unlocks a gate and prevents it from closing again). A Rest URL (resturl) is also extracted, which is required in the GET request for certain entity tables. This will be explained in greater detail later on in this documentation. Please note that the authentication tokens listed in the previous section are required by this script to extract the BHtoken.

Outcome#

  • This script extracts the BHtoken and restUrl and stores it in a local .txt file called tokens.txt.
